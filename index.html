🖋 BOOTHTRACKER
🖋 Booth Tracker 使用說明

🗂 兩個主頁面

頂部導覽列有兩個頁面：
廠商管理 — 管理你追蹤的音樂祭、市集主辦方
擺攤機會 — 爬蟲自動抓到的或你手動加入的攤位徵招資訊

☑ 選取模式（批次操作）

點右上角 ☑ 選取 進入選取模式，可以勾選多張廠商卡片，然後批次 設為追蹤 / 取消追蹤 / 刪除。再次點擊或按「取消」離開。

🔍 搜尋 + 排序

廠商管理頁有搜尋欄，可輸入名稱或標籤關鍵字即時篩選。右側排序下拉可依最新加入 / 名稱 / 最近更新 / 機會數量排序。

🔄 資料同步

資料存在 Firebase 雲端，多裝置開同一個網址就能即時共享。後端爬蟲每 8 小時自動跑一次（台灣時間早上 8 點、下午 4 點、午夜）。

手動新增的機會和爬蟲抓到的機會都存在同一個資料庫，不會互相覆蓋。

➕ 新增廠商

點右上角 ＋ 新增廠商，填入名稱即可。其他欄位：
社群連結 — 純顯示用，FB / IG 連結按鈕會出現在卡片上
FB 粉專 ID（爬蟲設定）— 爬蟲用這個欄位去抓 FB 貼文
標籤 / 備注 — 可搜尋的輔助資訊

⭐ 追蹤 / 取消追蹤

卡片左上角的 追蹤中 / 追蹤 按鈕控制「這個廠商是否被爬蟲監控」。
只有標為 ⭐ 追蹤中 的廠商，後端爬蟲才會去抓它的網站和 FB 貼文。

剛加入的廠商預設未追蹤，記得開啟。也可以用選取模式批次開啟。

📥 匯入範本

點搜尋欄右邊的 📥 匯入範本，可一次載入 10 個台灣常見音樂祭和市集（大港開唱、春浪、168幸福市集等）。已存在的不會重複匯入。

🔢 機會數量徽章

每張廠商卡片右上角顯示「X 個機會」，是這個廠商目前有效（未過期）的擺攤機會數量。點數字旁的標記可看 🆕 7天內新增的機會。

🤖 自動抓取 vs ✏️ 手動新增

機會有兩種來源：
🤖 自動抓取 — 爬蟲解析廠商官網或 FB 貼文後，由 Claude AI 提取出來
✏️ 手動 — 你自己點「＋ 手動新增」填入的

📋 狀態管理

每筆機會有狀態，可在機會卡片右側的下拉選單修改：
🆕 新發現 → 有興趣 → 已申請 → 已確認 / 未錄取
頁面頂部可依狀態篩選，讓你快速看到「已申請待確認」的項目。

⏰ 截止日顏色說明

機會卡片右上角的截止日用顏色提示緊迫程度：
14天以上 充裕  14天內 快到期  7天內/已過 緊急/過期

👻 顯示 / 隱藏已過期

預設隱藏已過截止日的機會，避免干擾。點右上角 顯示已過期（N） 可切換。

過期機會不會自動刪除，保留作為歷史紀錄。

📊 統計列

頁面上方四格統計：有效機會總數 / 7 天內新增 / 已申請 / 已確認。快速掌握目前的申請狀況。

🌐 官方網站爬蟲

編輯廠商 → 在「社群連結」填入 官方網站 URL，爬蟲每次執行時會爬取該網頁全文，交由 Claude AI 判斷是否有攤位招募資訊。

網站若需登入或是純 JS 渲染，爬蟲可能抓不到，這時依賴 FB 粉專或手動填寫。

📘 FB 粉專爬蟲

編輯廠商 → 在「爬蟲設定」的 FB 粉專 ID 填入 Facebook 粉專網址的最後一段文字。
例：https://www.facebook.com/dakangfest → 填 dakangfest
爬蟲透過 Facebook Graph API 抓取最新貼文，篩選含「招募、攤位、申請」等關鍵字的貼文後讓 AI 解析。

📷 IG 不支援自動抓取

IG 填入的帳號只用來顯示連結按鈕（點擊後開新分頁），Meta 的 API 限制無法自動讀取別人的 IG 內容。

⏱ 爬蟲執行時間

後端每 8 小時跑一次，UTC 時間的 0 */8 * * *，對應台灣時間 早上 8 點、下午 4 點、凌晨 12 點。

首次新增廠商後，等到下一個整點跑完就會看到機會。若想立即測試可到 Railway 手動 Deploy。

❌ 抓取異常

廠商卡片底部若顯示紅色「❌ 抓取異常」，代表上次爬蟲執行時這個廠商的網站或 FB 無回應。原因可能是：網站暫時下線、FB 粉專 ID 填錯、或 API Token 失效。
